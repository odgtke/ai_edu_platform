## 部署前准备

### 1.1 硬件要求
| 组件 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2核 | 4核以上 |
| 内存 | 4GB | 8GB以上 |
| 存储 | 50GB | 100GB以上 |
| 网络 | 100Mbps | 1Gbps |

### 1.2 软件依赖
- **操作系统**：CentOS 7.6+/Ubuntu 18.04+
- **Java**：OpenJDK 8 或 Oracle JDK 8
- **数据库**：MySQL 8.0
- **缓存**：Redis 6.x
- **Web服务器**：Nginx 1.18+

### 1.3 端口规划
| 服务 | 端口 | 用途 |
|------|------|------|
| 应用服务 | 8080 | Spring Boot应用端口 |
| MySQL | 3306 | 数据库服务 |
| Redis | 6379 | 缓存服务 |
| Nginx | 80/443 | Web服务 |

### 3.4 创建启动脚本
```bash
# 创建启动脚本
cat > /opt/edu-platform/start.sh << 'EOF'
#!/bin/bash

APP_NAME="ai-edu-platform"
APP_JAR="ai-edu-platform-1.0.0.jar"
APP_HOME="/opt/edu-platform"
CONFIG_DIR="$APP_HOME/config"
LOG_DIR="$APP_HOME/logs"

# 检查应用是否已在运行
check_running() {
    PID=$(ps -ef | grep $APP_JAR | grep -v grep | awk '{print $2}')
    if [ -n "$PID" ]; then
        echo "$APP_NAME is already running (PID: $PID)"
        return 1
    else
        return 0
    fi
}

# 启动应用
start_app() {
    if check_running; then
        echo "Starting $APP_NAME..."
        nohup java -jar $APP_HOME/$APP_JAR \
            --spring.config.location=$CONFIG_DIR/ \
            --spring.profiles.active=prod \
            > $LOG_DIR/startup.log 2>&1 &
        
        sleep 5
        if check_running; then
            echo "Failed to start $APP_NAME"
            return 1
        else
            echo "$APP_NAME started successfully"
            return 0
        fi
    fi
}

# 停止应用
stop_app() {
    PID=$(ps -ef | grep $APP_JAR | grep -v grep | awk '{print $2}')
    if [ -n "$PID" ]; then
        echo "Stopping $APP_NAME (PID: $PID)..."
        kill $PID
        sleep 5
        if ps -p $PID > /dev/null; then
            echo "Force killing $APP_NAME..."
            kill -9 $PID
        fi
        echo "$APP_NAME stopped"
    else
        echo "$APP_NAME is not running"
    fi
}

# 重启应用
restart_app() {
    stop_app
    sleep 2
    start_app
}

# 查看状态
status_app() {
    PID=$(ps -ef | grep $APP_JAR | grep -v grep | awk '{print $2}')
    if [ -n "$PID" ]; then
        echo "$APP_NAME is running (PID: $PID)"
        echo "Uptime: $(ps -o etime= -p $PID)"
    else
        echo "$APP_NAME is not running"
    fi
}

case "$1" in
    start)
        start_app
        ;;
    stop)
        stop_app
        ;;
    restart)
        restart_app
        ;;
    status)
        status_app
        ;;
    *)
        echo "Usage: $0 {start|stop|restart|status}"
        exit 1
        ;;
esac
EOF

chmod +x /opt/edu-platform/start.sh
```

### 3.5 配置系统服务
```bash
# 创建systemd服务文件
sudo cat > /etc/systemd/system/edu-platform.service << EOF
[Unit]
Description=智慧教育平台服务
After=network.target mysql.service redis.service

[Service]
Type=forking
User=edu-user
WorkingDirectory=/opt/edu-platform
ExecStart=/opt/edu-platform/start.sh start
ExecStop=/opt/edu-platform/start.sh stop
ExecReload=/opt/edu-platform/start.sh restart
Restart=on-failure
RestartSec=30

[Install]
WantedBy=multi-user.target
EOF

# 重新加载systemd配置
sudo systemctl daemon-reload

# 设置开机自启
sudo systemctl enable edu-platform
```

### 4.2 Nginx配置
```bash
# 创建Nginx配置文件
sudo cat > /etc/nginx/conf.d/edu-platform.conf << EOF
upstream edu_backend {
    server localhost:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name edu.yourdomain.com;
    
    # 日志配置
    access_log /var/log/nginx/edu_access.log;
    error_log /var/log/nginx/edu_error.log;
    
    # 客户端最大请求体大小
    client_max_body_size 10M;
    
    # API代理配置
    location /api/ {
        proxy_pass http://edu_backend/api/;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
        
        # 超时配置
        proxy_connect_timeout 30s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }
    
    # 前端静态文件配置（如果有的话）
    location / {
        root /opt/edu-platform/frontend;
        index index.html;
        try_files \$uri \$uri/ /index.html;
    }
    
    # 健康检查
    location /health {
        access_log off;
        return 200 "healthy\\n";
        add_header Content-Type text/plain;
    }
}
EOF

# 测试Nginx配置
sudo nginx -t

# 重载Nginx配置
sudo systemctl reload nginx
```

```bash
    echo "=== JVM Status ==="
```

### 6.3 自动化监控
```bash
# 创建健康检查脚本
cat > /opt/edu-platform/scripts/health_check.sh << 'EOF'
#!/bin/bash

APP_URL="http://localhost:8080/api/actuator/health"
NOTIFY_EMAIL="admin@yourdomain.com"

# 检查应用健康状态
response=$(curl -s -o /dev/null -w "%{http_code}" $APP_URL)

if [ "$response" != "200" ]; then
    echo "$(date): Application health check failed with status $response" | \
        mail -s "智慧教育平台告警" $NOTIFY_EMAIL
    # 重启服务
    sudo systemctl restart edu-platform
fi
EOF

chmod +x /opt/edu-platform/scripts/health_check.sh

# 添加到crontab
echo "*/5 * * * * /opt/edu-platform/scripts/health_check.sh" | crontab -
```

### 7.2 应急处理流程
1. **故障发现**：监控告警或用户反馈
2. **初步诊断**：查看日志和系统状态
3. **临时处理**：重启服务或切换备用节点
4. **根本解决**：分析原因并修复问题
5. **验证恢复**：确认服务恢复正常
6. **总结改进**：记录故障和预防措施

## 升级维护

```bash

# 2. 部署新版本
cd /opt/edu-platform

mv ai-edu-platform-1.1.0.jar ai-edu-platform.jar

# 3. 更新配置文件（如有变更）
# cp /opt/edu-platform.backup/config/* /opt/edu-platform/config/

# 4. 启动新版本
sudo systemctl start edu-platform

# 5. 验证升级结果
curl http://localhost/api/version
```

### 8.2 数据备份策略
```bash
# 创建备份脚本
cat > /opt/edu-platform/scripts/backup.sh << 'EOF'
#!/bin/bash

BACKUP_DIR="/opt/edu-platform/backup"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR/$DATE

# 数据库备份
mysqldump -u edu_user -p edu_platform > $BACKUP_DIR/$DATE/database.sql

# 应用配置备份
tar -czf $BACKUP_DIR/$DATE/config.tar.gz /opt/edu-platform/config

# 应用日志备份
tar -czf $BACKUP_DIR/$DATE/logs.tar.gz /opt/edu-platform/logs

# 清理7天前的备份
find $BACKUP_DIR -type d -mtime +7 -exec rm -rf {} \;
EOF

chmod +x /opt/edu-platform/scripts/backup.sh

# 设置定时备份
echo "0 2 * * * /opt/edu-platform/scripts/backup.sh" | crontab -
```

### 8.3 回滚方案

```bash
# 回滚到备份版本
sudo systemctl stop edu-platform

# 恢复备份
rm -rf /opt/edu-platform
cp -r /opt/edu-platform.backup.20260311 /opt/edu-platform

# 重启服务
sudo systemctl start edu-platform
```
