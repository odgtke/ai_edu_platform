# 前端白屏问题排查与修复流程

## 问题描述

智慧教育平台前端页面显示白屏，无法正常展示内容。

---

## 问题排查流程

### 第一阶段：服务状态检查

#### 1.1 检查前后端服务进程
```powershell
# 检查前端服务 (Node.js)
tasklist | findstr "node"

# 检查后端服务 (Java)
tasklist | findstr "java"
```

#### 1.2 检查端口监听状态
```powershell
# 检查 3000 端口（前端）
netstat -ano | findstr ":3000" | findstr "LISTENING"

# 检查 8081 端口（后端）
netstat -ano | findstr ":8081" | findstr "LISTENING"
```

#### 1.3 验证服务可访问性
```powershell
# 测试前端服务
Invoke-WebRequest -Uri http://localhost:3000/ -UseBasicParsing -TimeoutSec 5

# 测试后端服务
Invoke-WebRequest -Uri http://localhost:8081/actuator/health -UseBasicParsing -TimeoutSec 5
```

**常见问题**：
- 端口被占用但进程已不存在（僵尸端口）
- 服务启动后自动崩溃
- 防火墙阻止访问

---

### 第二阶段：浏览器控制台分析

#### 2.1 打开浏览器开发者工具
- 按 `F12` 或 `Ctrl+Shift+I` 打开开发者工具
- 切换到 **Console** 控制台标签

#### 2.2 分析错误类型

| 错误类型 | 示例 | 可能原因 |
|---------|------|---------|
| `ReferenceError` | `Lightbulb is not defined` | 组件/变量未导入 |
| `SyntaxError` | `does not provide an export named 'xxx'` | 导入的模块不存在该导出 |
| `TypeError` | `Cannot read property of undefined` | 空值访问 |
| `Network Error` | `Failed to fetch` | 网络请求失败 |

---

### 第三阶段：代码级问题定位

#### 3.1 检查 main.js 入口文件
```javascript
// 确认 Vue 应用正确创建和挂载
import { createApp } from 'vue'
import App from './App.vue'

const app = createApp(App)
app.mount('#app')
```

#### 3.2 检查路由配置
```javascript
// router/index.js
import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),  // 开发环境建议使用 hash 模式
  routes
})
```

**关键决策**：
- `createWebHistory()`：需要服务器配置 history fallback
- `createWebHashHistory()`：无需服务器配置，适合开发和简单部署

#### 3.3 检查组件导入
```javascript
// 错误示例：导入不存在的图标
import { Lightbulb } from '@element-plus/icons-vue'  // ❌ 不存在

// 正确做法：使用存在的图标
import { Opportunity } from '@element-plus/icons-vue'  // ✓ 存在
```

#### 3.4 验证 Element Plus 图标可用性
```powershell
# 列出所有可用图标
cd frontend
node -e "const icons = require('@element-plus/icons-vue'); console.log(Object.keys(icons).join(', '))"

# 搜索特定图标
cd frontend
node -e "const icons = require('@element-plus/icons-vue'); console.log(Object.keys(icons).filter(k => k.toLowerCase().includes('light')).join(', '))"
```

## 本次问题修复记录

### 问题 1：Element Plus 图标未正确注册

**现象**：动态图标组件无法解析

**修复方案**：在 main.js 中全局注册所有图标
```javascript
import * as ElementPlusIconsVue from '@element-plus/icons-vue'

const app = createApp(App)

// 注册所有 Element Plus 图标
for (const [key, component] of Object.entries(ElementPlusIconsVue)) {
  app.component(key, component)
}
```

### 问题 2：路由模式配置不当

**现象**：页面刷新后 404

**修复方案**：使用 hash 模式
```javascript
// 修改前
history: createWebHistory()

// 修改后
history: createWebHashHistory()
```

### 问题 3：图标不存在导致运行时错误

**现象**：`Uncaught ReferenceError: Lightbulb is not defined`

**根因**：`@element-plus/icons-vue` 中没有 `Lightbulb` 图标

**修复方案**：
1. 查找可用替代图标：`Opportunity`（灵感/机遇）
2. 更新导入语句
3. 更新组件注册

```javascript
// 修改前
import { Lightbulb } from '@element-plus/icons-vue'
components: { ..., Lightbulb, ... }

// 修改后
import { Opportunity } from '@element-plus/icons-vue'
components: { ..., Opportunity, ... }
```

## 预防措施

### 1. 图标使用规范

#### 1.1 使用前验证图标存在性
```powershell
# 快速验证图标是否存在
cd frontend
node -e "const icons = require('@element-plus/icons-vue'); console.log('Lightbulb' in icons ? '存在' : '不存在')"
```

#### 1.2 建立图标白名单
在项目中维护一个已验证的图标列表，避免使用不存在图标。

### 2. 组件导入规范

#### 2.1 使用 ESLint 检查
配置 ESLint 规则检测未使用的导入和未定义的变量。

#### 2.2 导入时验证
```javascript
// 推荐：显式列出所有导入
import {
  Timer,
  Calendar,
  DataAnalysis,
  // ... 其他图标
} from '@element-plus/icons-vue'
```

### 3. 路由配置规范

| 环境 | 推荐模式 | 说明 |
|-----|---------|------|
| 开发环境 | `createWebHashHistory` | 无需服务器配置 |
| 生产环境（Nginx） | `createWebHistory` + fallback 配置 | 需要配置 try_files |
| 生产环境（静态托管） | `createWebHashHistory` | 简单部署 |

### 4. 全局错误处理

在 main.js 中添加全局错误处理：
```javascript
app.config.errorHandler = (err, vm, info) => {
  console.error('Vue Error:', err)
  console.error('Component:', vm)
  console.error('Info:', info)
}
```

## 快速诊断清单

当遇到白屏问题时，按以下顺序检查：

- [ ] 1. 服务是否运行（`netstat` 检查端口）
- [ ] 2. 浏览器控制台是否有错误
- [ ] 3. 网络请求是否正常（Network 标签）
- [ ] 4. main.js 是否正确挂载应用
- [ ] 5. 路由配置是否正确
- [ ] 6. 组件导入是否完整
- [ ] 7. 第三方库版本是否兼容

## 总结

本次白屏问题的根本原因是 **Element Plus 图标库中不存在 `Lightbulb` 图标**，导致运行时抛出 `ReferenceError`，Vue 应用无法正常渲染。

**关键教训**：
1. 使用第三方库的组件/图标前，务必确认其存在性
2. 开发时应开启浏览器控制台，及时发现运行时错误
3. 全局注册常用组件可以减少局部导入遗漏的问题
4. 使用 Hash 路由模式可以简化开发环境配置
