# 前端 Vue 文件乱码修复流程文档

## 一、问题背景

### 1.1 问题现象
- 前端 Vue 文件出现大量中文乱码
- 页面显示为 GBK 编码被误读为 UTF-8 导致的乱码字符（如 `瀛︿範涓�績`、`鐢ㄦ埛绠＄悊` 等）
- 项目无法正常编译启动，Vite 报错 "Element is missing end tag"、"Attribute name cannot contain..."

### 1.2 问题原因
- 文件创建时使用了错误的编码格式（GBK）
- 编辑器以 UTF-8 格式读取 GBK 编码文件，导致中文显示为乱码
- 乱码破坏了 Vue 模板语法结构（引号、标签等），导致编译失败

## 二、修复策略

### 2.1 核心原则
1. **先核心后次要**：优先修复核心功能页面，确保系统可用
2. **渐进式修复**：从关键路径开始，逐步扩展
3. **备份优先**：修复前备份原始文件，防止数据丢失
4. **验证驱动**：每修复一批文件立即验证，确保无回归

### 2.2 修复优先级
```
P0 - 核心页面（必须修复）
```

## 三、修复流程

### 3.1 准备阶段

#### 3.1.1 备份原始文件
```powershell
cd frontend\src\views
New-Item -ItemType Directory -Force -Path backup
Copy-Item *.vue backup\
Copy-Item analytics\*.vue backup\
```

#### 3.1.2 评估损坏范围
```powershell
# 检查所有 Vue 文件的编码状态
Get-ChildItem *.vue | ForEach-Object {
    $content = Get-Content $_.FullName -Raw -Encoding UTF8
    if ($content -match '[\u4e00-\u9fa5]') {
        Write-Host "$($_.Name): 包含正常中文"
    } else {
        Write-Host "$($_.Name): 可能损坏"
    }
}
```

## 四、技术要点

### 4.1 编码规范

#### 4.1.1 文件编码要求
- 所有 Vue 文件必须使用 **UTF-8 without BOM** 编码
- 禁止混用 GBK/GB2312 等中文编码
- 编辑器统一设置：File Encoding = UTF-8

#### 4.1.2 中文内容书写规范
```vue
<!-- 正确示例 -->
<template>
  <div class="page-header">
    <h1>数据概览</h1>
    <p>实时监控平台核心数据指标</p>
  </div>
</template>

<!-- 错误示例（乱码） -->
<template>
  <div class="page-header">
    <h1>鏁版嵁姒傝堣</h1>  <!-- 乱码！ -->
    <p>瀹炴椂鐩戞帶骞冲彴鏍稿績鏁版嵁鎸囨爣</p>  <!-- 乱码！ -->
  </div>
</template>
```

### 4.2 常见错误处理

#### 4.2.1 Vite 编译错误
```
[vite] Pre-transform error: Element is missing end tag.
[vite] Pre-transform error: Attribute name cannot contain U+0022 ("), U+0027 ('), and U+003C (<).
```
**原因**：乱码破坏了 HTML 标签结构
**解决**：重新创建文件，确保编码正确

#### 4.2.2 引号不匹配
```vue
<!-- 错误 -->
<el-button type="primary>确定</el-button>

<!-- 正确 -->
<el-button type="primary">确定</el-button>
```

### 4.3 工具使用

#### 4.3.1 create_file 工具
- 用于重新创建完整的 Vue 文件
- 自动使用 UTF-8 编码
- 适合严重乱码的文件重建

#### 4.3.2 search_replace 工具
- 用于修复局部乱码
- 需要确保原文本精确匹配
- 适合轻微乱码的修复

## 五、预防措施

### 5.1 开发规范
1. **统一编辑器设置**：所有开发人员使用 UTF-8 编码
2. **Git 配置**：设置 `core.autocrlf=false`，避免换行符转换问题
3. **文件模板**：创建新的 Vue 文件模板，确保编码正确

### 5.2 检查机制
```powershell
# 编码检查脚本
function Check-FileEncoding {
    param([string]$Path)
    $bytes = [System.IO.File]::ReadAllBytes($Path)
    if ($bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF) {
        return "UTF-8 with BOM"
    }
    # 检查是否包含 GBK 特征字节
    return "Unknown/UTF-8"
}
```

### 5.3 CI/CD 集成
在持续集成流程中添加编码检查步骤：
```yaml
- name: Check File Encoding
  run: |
    find src -name "*.vue" -exec file {} \; | grep -v "UTF-8" && exit 1 || echo "All files are UTF-8"
```
